Projeto Centelha

Arquitetura, stack tecnológica e fluxo de desenvolvimento da aplicação Laravel 12 + React 19 + Inertia + Vite.

Monorepo Docker Compose SPA via Inertia S3-Compatible Storage Queue Worker

Stack Tecnológica

🟥

Laravel 12

Backend PHP 8.4 com rotas, controllers, Eloquent, migrations, queues e seeders.

⚛️

React 19

Renderização declarativa dos componentes de página e interface do usuário.

🔗

Inertia.js

Ponte entre Laravel e React: SPA sem API REST explícita entre frontend e backend.

⚡

Vite

Build tool e HMR (Hot Module Replacement) na porta 5173.

🐘

PostgreSQL 16

Banco de dados relacional principal em desenvolvimento e produção.

🪣

MinIO

Armazenamento S3-compatible para uploads e arquivos da aplicação.

🐳

Docker Compose

Ambiente de desenvolvimento e produção containerizados.

🎨

Tailwind + shadcn/ui

Estilização utilitária com componentes base reutilizáveis.

Fluxo Inertia.js

1. Requisição Usuário clica em link/button do React
→
2. Laravel Rota → Controller → Responde com JSON de página
→
3. Inertia Troca apenas o componente React ativo
→
4. SPA feel Navegação rápida sem reload completo

Comandos Essenciais

Subir ambiente de desenvolvimento

./setup.sh && docker compose up -d --build

Migrations + seeds

docker exec centelha-php php artisan migrate --seed

Executar testes

docker exec centelha-php php artisan test

Lint / format frontend

npm run check

Formatar PHP

docker exec centelha-php vendor/bin/pint

Worker de filas

docker exec centelha-php php artisan queue:work --tries=3 --timeout=90

Convenências Importantes

  • Use componentes shadcn/ui de resources/js/components/ui/.
  • Tokens de tema via CSS variables; evite cores Tailwind hard-coded.
  • Áreas de scroll horizontal devem esconder a barra: [&::-webkit-scrollbar]:hidden.
  • Rotas admin usam middleware role:admin; rotas autenticadas usam auth.
  • Prefira npm ao invés de pnpm (Docker e scripts usam package-lock.json).
  • Testes usam SQLite in-memory; não altere para PostgreSQL sem intenção.
